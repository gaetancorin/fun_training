Attention !
NOMUTILISATEUR doit être remplacé a la main par le vrai nom d'utilisateur
USERPROFILE est une variable d'environnement et ne doit pas être changé

---- INSTALLER UNE VERSION DE PYTHON ---
aller sur google et ecrire python3.11 ou python3.12
selectionner les liens python.org ayant la plus haute version (3.11.9 et pas 3.11.0)
le lien doit avoir un fichier "Windows installer (64-bits)", telecharger le fichier d'installation

lors du double clique d'installation du fichier:
NE PAS cocher "add python.exe to PATH"
faire "customize installation"
optional features -> laisser tout cocher
Advanced options -> changer l'url a C:\python311
Advanced options -> laisser cocher "Associate files…", "Create shortcuts…", "Add python to environnement variables"

---- INSTALLER PYENV ----
sur powershell en administrateur
autoriser les scripts utilisteur en ecrivant dans le terminal:
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

ouvrir un terminal classique
installer pyenv:
pip install pyenv-win --target $env:USERPROFILE\.pyenv

sur windows, aller dans "modifier les variables d'environnement systèmes"
cliquer sur "variables d'environnement"
dans "variables utilisateur pour NOMUTILISATEUR", double cliquer sur "Path"
ajouter les 2 nouvelles variables (elles doivent être tout en haut de la liste)
C:\Users\NOMUTILISATEUR\.pyenv\pyenv-win\shims
C:\Users\NOMUTILISATEUR\.pyenv\pyenv-win\bin

relancer le terminal, ecrire:
pyenv --version
si l'installation est bien faite, cela devrais dire que les variables d'environnements
PYENV, PYENV_ROOT et PYENV_HOME ne sont pas installé

sur windows, aller dans "modifier les variables d'environnement systèmes"
cliquer sur "variables d'environnement"
dans "variables utilisateur pour NOMUTILISATEUR" ajouter 3 variables 
PYENV		C:\Users\NOMUTILISATEUR\.pyenv\pyenv-win
PYENV_ROOT	C:\Users\NOMUTILISATEUR\.pyenv\pyenv-win
PYENV_HOME	C:\Users\NOMUTILISATEUR\.pyenv\pyenv-win

relancer le terminal. Pour chaque version de python installé, copier ces versions dans pyenv avec :
Copy-Item -Recurse "C:\Python311" "$env:USERPROFILE\.pyenv\pyenv-win\versions\3.11"

voir les versions python installé sur pyenv:
pyenv versions

supprimer toutes les variables d'environnement qui pointe vers un python spécifique
sur windows, aller dans "modifier les variables d'environnement systèmes"
cliquer sur "variables d'environnement"
supprimer les lignes suivantes dans toutes les variables d'environnements (notmalement dans le path utilisateurs et le path de variable systèmes)
C:\Python311\Scripts\
C:\Python311\

relancer le terminal, puis ecrire:
pyenv rehash
python --version
pyenv global 3.12
python --version
pyenv global 3.11
python --version

Installation terminé
voir version = pyenv versions
changer version= pyenv global 3.11

------------------
Supprimer proprement une version Python
Aller dans paramètres -> appplication -> application installés
écrire python3.11 dans la barre de recherche et cliquer sur désinstaller
puis supprimer manuellement le dossier "C:/python3.11" qui est désormais presque vide

-----------------
Une betise sur l'installation/la desinstallation ?
telecharger la version python sur python.org, et lancer l'exécutable pour réparer
puis désinstaller proprement si nécessaire comme expliqué

--------------
Problème de pip ?
écrire sur le terminal:
pip --version
python -m pip --version
pip mène vers C:/pythonXX
python -m pip mène vers pyenv
Il n'y a malheureusement pas de moyen de réparer ca.
Il faut donc faire les installation en réalisant:
python -m pip
